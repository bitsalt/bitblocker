## One-liner

Self-hosted single-binary Go daemon that drops inbound traffic from selected countries at the Traefik edge via a `forwardAuth` middleware, for operators running small-scale self-hosted infrastructure.

## Stack

- **Language:** Go 1.22.2 (toolchain pinned in `go.mod`).
- **Framework:** standard library + `oschwald/maxminddb-golang v1.13.1` (MMDB reader, pinned below the Go 1.24 floor v2 introduced); `maxmind/mmdbwriter v1.0.0` (test-only); `gopkg.in/yaml.v3`; `stretchr/testify`.
- **Tests:** `go test -race ./...`.
- **Build / package manager:** Go modules + `Makefile` (wraps `go build`, `go test`, `golangci-lint run`, `govulncheck`).
- **Deploy:** none yet — Sprint 4 ships multi-stage Dockerfile, systemd unit, and a GHA release workflow producing linux/amd64 + linux/arm64 binaries. No deploy substrate exists today.

## Key invariants

- **Fail-closed posture is load-bearing.** Cold start with no ruleset default-denies. Disk cache, `behavior.startup_mode: fail-closed | fail-open` knob, `/healthz` 503 while empty, and a bounded retry budget make it operationally tolerable. See decisions log 2026-04-22.
- **Header selection is `X-Real-IP` first, then rightmost-XFF.** Leftmost-XFF is spoofable under Traefik's `trustForwardHeader: true`. Malformed `/check` fails closed. Decisions log 2026-04-22.
- **Atomic swap only.** Blocklist updates are pointer-swap under `RWMutex`; partial or corrupted updates never reach lookup. Spec § Blocklist update strategy.
- **MMDB binary, not CSV.** MaxMind data is consumed via `maxminddb-golang`. Decisions log 2026-04-22.
- **IPv4 + IPv6 are first-class** (against the original spec's deferred-IPv6 stance). The trie has separate v4/v6 roots on `net/netip`; `Contains` normalizes IPv4-in-IPv6 via `Unmap`.
- **MaxMind license key may come from `MAXMIND_LICENSE_KEY` env or inline YAML; env wins.** Prefer env so the secret does not sit on disk.
- **Toolchain pin is deliberate.** Do not bump `go.mod`'s `go 1.22.2` or the MMDB library versions without resolving the toolchain-bump Open Question first — `maxminddb-golang/v2` introduced a Go 1.24 floor.
- **Repo classification: production-touching.** All changes go via feature branch + PR per `vault-CLAUDE.md`. `gh pr create` after push.
- **License: MIT** (decisions log 2026-04-23). Relicensing is possible later if the project heads toward a CNCF/Apache-umbrella home.
- **`golangci-lint` pinned to v2.11.4 in CI**; local installs run whatever `go install @latest` resolves. If local and CI diverge, bump the CI pin.
- **Pre-commit is raw shell, not the `pre-commit` framework.** Solo-Go project; no Python toolchain pulled in for one linter.
